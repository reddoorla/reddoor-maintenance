# Blux Pipeline — Plan 1: Target Slices Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the four Prismic slices the Blux conversion pipeline emits into — Hero, MediaText, Section/Grid, CollectionList — to `reddoor-starter`, each rendering correctly and passing axe.

**Architecture:** Filesystem-first Slice Machine slices (`src/lib/slices/<Name>/{model.json,index.svelte,mocks.json}`) following the existing `RichText` slice pattern. Prismic slice zones are **flat** (slices cannot nest), so Section/Grid holds a repeatable `items` group rather than child slices, and CollectionList receives its resolved documents via SliceZone `context` (the page load fetches them). Components use Svelte 5 runes + Tailwind, matching the starter.

**Tech Stack:** SvelteKit, Svelte 5 (runes), `@slicemachine/adapter-sveltekit`, `@prismicio/client`, Tailwind v4, Vitest + `@testing-library/svelte`, Playwright axe (`tests/a11y.spec.ts`).

**Repo:** all work in `reddoor-starter`. This plan doc lives with the spec in `reddoor-maintenance` for cohesion.

---

## File structure

Per slice, created under `reddoor-starter/src/lib/slices/<Name>/`:
- `model.json` — Prismic SharedSlice schema (the migration target shape).
- `index.svelte` — the render component (Svelte 5 runes; `slice: Content.<Name>Slice` prop).
- `mocks.json` — Slice Machine mock data (drives the a11y fixture + simulator).
- `<Name>.test.ts` — Vitest render test (colocated, `*.test.ts`).

Shared, modified once:
- `src/lib/slices/index.js` — the generated registry; regenerated by Slice Machine, verified to import all five slices.
- `src/routes/dev/a11y-fixtures/+page.svelte` — add each new slice so `tests/a11y.spec.ts` (axe) covers it.

Reused as-is: `src/lib/components/HeroBackgroundImage.svelte`, `RichTextBody.svelte`, `Img.svelte`, `src/lib/utils/image.ts`.

**Slice registry note:** `src/lib/slices/index.js` is marked "Code generated by Slice Machine. DO NOT EDIT." After creating each slice's `model.json`, run `pnpm exec slicemachine` once (or `pnpm run slicemachine`) to regenerate the registry; if the codegen is unavailable in the environment, hand-add the `import` + `components` entry (it is regenerated identically on the next Slice Machine run). Each task's registry step below shows the exact resulting content.

## Task 4: CollectionList slice

**Modeling note:** A slice cannot query Prismic on its own. CollectionList declares which repeatable type it renders (`collection_type` text = the custom-type api-id, e.g. `product`) and a `display` variation; the **page load fetches the documents** and passes them to the SliceZone as `context={{ collections }}`. The component reads `context.collections[slice.primary.collection_type]`. This keeps the slice deterministic and SSR-friendly.

## Execution notes (2026-07-05) — deviations applied during the build

Executed on branch `blux-slices` in reddoor-starter (off main). The plan-as-written
had these issues; all were fixed during the build and are the source of truth for a
re-run:

- **Headings render via `PrismicRichText`, not `RichTextBody`.** `RichTextBody`
  normalizes/floors heading aria-levels for heading-order a11y — wrong for a
  controlled single-heading field, and it makes the `level: 1`/`level: 3` test
  assertions unsatisfiable. Applies to every slice `heading`, SectionGrid
  `item_heading`, and the CollectionList doc title; body prose still uses `RichTextBody`.
- **Hero gained `bg-neutral-900`** so the white heading keeps contrast when the
  background image is absent/slow (and to pass axe over the hermetic pixel fixture).
- **Image test mocks need `dimensions`** — `PrismicImage` reads `field.dimensions.width`;
  the SectionGrid/CollectionList item-media mocks were missing it.
- **CollectionList's second test is `container`-scoped** — `@testing-library/svelte`
  doesn't auto-clean between tests here, so a document-wide `getAllByRole` also counted
  the prior test's render.
- **Types + registry hand-authored** — the packaged Slice Machine is UI-only
  (`start-slicemachine`; no headless codegen), so `src/lib/slices/index.js` and the
  `Content.*Slice` types in **`src/prismicio-types.d.ts`** (note: `src/`, not `src/lib/`)
  were hand-written. Both regenerate canonically on the next Slice Machine run.
- **`as unknown as Content.*Slice`** replaced `as any` mocks (repo lint bans explicit `any`).
- Two implementer subagents died on transient infra errors (0 tool calls); Tasks 2–5
  were completed inline.
