# `@reddoorla/maintenance` 0.8.0 — Workflow Closure Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the operator workflow loop opened in 0.7.0. Eliminate the manual `audit lighthouse → Websites` paste. Pay down the test debt on reports orchestrators. Clean up dead code from 0.7.0. Vendor away the CloudFront image dependency.

**Prerequisite:** 0.7.0 (the report concept) must be merged to `main`. This plan branches off `main` after that merge. See [MORNING_REPORT_2026-05-27.md](../../../MORNING_REPORT_2026-05-27.md) for the deep-review context that drove this scope.

**Architecture:**

- **Closing the lighthouse → Websites loop** means a new flag on `audit lighthouse`: `--write-airtable [slug]`. The audit produces scores; the new flag pushes them into the matching Websites row's `pScore` / `rScore` / `bpScore` / `seoScore` fields + an `Last lighthouse audit at` timestamp (new field, added in Phase 0). Slug auto-resolves from cwd's `package.json#name` if not provided.
- **Airtable as inventory** means a new `InventoryProvider` factory `fromAirtableBase()` parallel to the existing `fromJsonFile()` ([src/inventory/json.ts:33-38](../../src/inventory/json.ts)). CLI: `--fleet airtable` resolves to `fromAirtableBase(openBase(readAirtableConfig()))`. Eliminates the JSON inventory file for Reddoor's day-to-day use.
- **Reports orchestrator tests** use a thin typed `Pick<AirtableBase, ...>` interface so we never have to fake the full Airtable SDK surface. The pattern mirrors `AuditContext` ([src/audits/util/inject.ts](../../src/audits/util/inject.ts)) — orchestrators accept an injectable `base` (already do), tests pass a stub.
- **Dead-code cleanup** + **CloudFront vendoring** are tactical 0.7.0 follow-ups bundled into the same minor for narrative coherence.
- **Reports module continues to live under `src/reports/`** — no architectural changes.

**Tech Stack:** No new runtime deps. Vendoring images into `src/reports/maintenance-email/assets/` requires a tsup config touch (`loader: { ".png": "binary", ".jpg": "binary" }` or move the images out of the bundle entirely and require fs reads at runtime).

Append to `src/reports/airtable/attachments.ts`:

```ts
/**
 * Airtable attachments can be created from a public URL OR uploaded directly via the
 * content.airtable.com "upload attachment" endpoint, which accepts base64 — we use that
 * because we don't have anywhere public to host generated HTML.
 * Docs: https://airtable.com/developers/web/api/upload-attachment
 */
export async function uploadAttachment(
  recordId: string,
  fieldName: string,
  bytes: Uint8Array | string,
  filename: string,
  contentType: string,
): Promise<void> {
  const apiKey = process.env.AIRTABLE_PAT;
  const baseId = process.env.AIRTABLE_BASE_ID;
  if (!apiKey || !baseId) {
    throw new Error("AIRTABLE_PAT and AIRTABLE_BASE_ID must be set");
  }
  const base64 = typeof bytes === "string" ? Buffer.from(bytes, "utf-8").toString("base64") : Buffer.from(bytes).toString("base64");
  const body = { contentType, file: base64, filename };
  const fieldPath = encodeURIComponent(fieldName);
  const url = `https://content.airtable.com/v0/${baseId}/${recordId}/${fieldPath}/uploadAttachment`;
  const res = await fetch(url, {
    method: "POST",
    headers: {
      Authorization: `Bearer ${apiKey}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify(body),
  });
  if (!res.ok) {
    throw new Error(`Airtable upload failed: ${res.status} ${res.statusText} ${await res.text()}`);
  }
}
```

Note the generalization: it now takes `fieldName` so it can be reused for any attachment field (not just `Rendered HTML`).

## Task 2.1: Fake AirtableBase helper

```ts
// tests/reports/_helpers/fake-airtable-base.ts
import type { AirtableBase } from "../../../src/reports/airtable/client.js";

/**
 * Calls captured by the fake. Tests assert against this.
 * Each entry records which table + method + payload.
 */
export type CapturedCall =
  | { kind: "select"; table: string; opts: Record<string, unknown> }
  | { kind: "create"; table: string; records: Array<{ fields: Record<string, unknown> }> }
  | { kind: "update"; table: string; records: Array<{ id: string; fields: Record<string, unknown> }> };

export type FakeRecord = { id: string; fields: Record<string, unknown> };

export type FakeAirtableBase = AirtableBase & {
  __calls: CapturedCall[];
  __records: Map<string, FakeRecord[]>;
};

/**
 * Build a fake AirtableBase. Seed `seed` maps table name → records that select() will return.
 * create() / update() append to __calls and (for create) push to __records[table].
 */
export function makeFakeBase(seed: Record<string, FakeRecord[]> = {}): FakeAirtableBase {
  const calls: CapturedCall[] = [];
  const records = new Map<string, FakeRecord[]>();
  for (const [table, rs] of Object.entries(seed)) records.set(table, [...rs]);

  const tableFn = (table: string) => {
    const tableRecords = records.get(table) ?? [];
    return {
      select: (opts: Record<string, unknown> = {}) => ({
        eachPage: async (
          cb: (recs: FakeRecord[], next: () => void) => void,
        ): Promise<void> => {
          calls.push({ kind: "select", table, opts });
          // Apply filterByFormula if provided — naive substring match against id.
          // For more complex filters tests should seed only matching rows.
          cb(tableRecords, () => {});
        },
      }),
      create: async (recs: Array<{ fields: Record<string, unknown> }>) => {
        calls.push({ kind: "create", table, records: recs });
        const created = recs.map((r, i) => ({ id: `rec_fake_${calls.length}_${i}`, fields: r.fields }));
        records.set(table, [...tableRecords, ...created]);
        return created;
      },
      update: async (recs: Array<{ id: string; fields: Record<string, unknown> }>) => {
        calls.push({ kind: "update", table, records: recs });
        // Apply the update to seeded records.
        for (const r of recs) {
          const idx = tableRecords.findIndex((x) => x.id === r.id);
          if (idx >= 0) {
            tableRecords[idx] = { id: r.id, fields: { ...tableRecords[idx].fields, ...r.fields } };
          }
        }
        return recs.map((r) => ({ id: r.id, fields: r.fields }));
      },
    };
  };

  const base = tableFn as unknown as FakeAirtableBase;
  base.__calls = calls;
  base.__records = records;
  return base;
}
```

import type { AuditResult } from "../types.js";
import type { LighthouseScores } from "../reports/types.js";

export function lighthouseScoresFromResult(result: AuditResult): LighthouseScores {
  if (result.audit !== "lighthouse") {
    throw new Error(`Expected a 'lighthouse' AuditResult, got '${result.audit}'`);
  }
  const summary = (result.details?.summary ?? {}) as Record<string, number>;
  const toPct = (n: number | undefined) =>
    typeof n === "number" && !Number.isNaN(n) ? Math.round(n * 100) : 0;
  return {
    performance: toPct(summary["performance"]),
    accessibility: toPct(summary["accessibility"]),
    bestPractices: toPct(summary["best-practices"]),
    seo: toPct(summary["seo"]),
  };
}

import { readFile } from "node:fs/promises";
import { join } from "node:path";
import { siteSlug } from "../reports/airtable/websites.js";

export async function resolveSlugFromCwd(cwd: string): Promise<string> {
  try {
    const pkgPath = join(cwd, "package.json");
    const raw = await readFile(pkgPath, "utf-8");
    const pkg = JSON.parse(raw) as { name?: string };
    if (!pkg.name) throw new Error("package.json has no 'name' field");
    return siteSlug(pkg.name);
  } catch (e) {
    throw new Error(
      `Could not derive site slug from ${cwd}/package.json: ${(e as Error).message}. ` +
        `Pass --write-airtable=<slug> explicitly.`,
    );
  }
}

```ts
// src/inventory/airtable.ts
import type { Site, InventoryProvider } from "../types.js";
import type { AirtableBase } from "../reports/airtable/client.js";
import { listWebsites, siteSlug } from "../reports/airtable/websites.js";

export type AirtableInventoryOptions = {
  /**
   * Where to assume sites are checked out. Each site's `path` is `{workdir}/{slug}`.
   * Defaults to the value of REDDOOR_FLEET_WORKDIR if set, else throws unless `path` is supplied.
   */
  workdir?: string;
};

/**
 * Read sites from the Airtable Websites table. Each Website row becomes one Site.
 * `path` is computed as `{workdir}/{slug}` since Airtable doesn't track local checkout paths.
 * Sites in status="deprecated" or status="probably not our problem" are excluded.
 */
export function fromAirtableBase(
  base: AirtableBase,
  opts: AirtableInventoryOptions = {},
): InventoryProvider {
  return async (): Promise<Site[]> => {
    const workdir = opts.workdir ?? process.env.REDDOOR_FLEET_WORKDIR;
    if (!workdir) {
      throw new Error(
        "fromAirtableBase requires `workdir` option or REDDOOR_FLEET_WORKDIR env (sites need a local path)",
      );
    }
    const websites = await listWebsites(base);
    return websites
      .filter((w) => w.maintenanceFreq !== "None" || w.testingFreq !== "None") // active sites only
      .map((w) => ({
        path: `${workdir}/${siteSlug(w.name)}`,
        name: siteSlug(w.name),
        repoUrl: w.url, // assumes url is the git URL — see note below
        meta: { airtableRowId: w.id, displayName: w.name },
      }));
  };
}
```

Note: this conflates "site URL" (the production URL) with `repoUrl` (the git URL). In Reddoor's Airtable today, `url` is the production URL. If you want this inventory provider to support cloning (the way JSON inventory does via `--workdir`), you need either: (a) add a `repoUrl` field to the Websites schema, or (b) derive the git URL via convention (e.g., `https://github.com/reddoor/{slug}`). **Flag this for the executor — they should ask Tucker which way to go before committing.**

## Notes for the executor

- **The `audit lighthouse` LHCI summary has fractional scores in [0,1].** `lighthouseScoresFromResult` multiplies by 100 + rounds; this matches what the report template expects (integer percentages).
- **The Airtable Websites field is named `pScore` / `rScore` / `bpScore` / `seoScore`** (lowercase, no spaces). Don't typo as `pscore` or `PScore`.
- **`--write-airtable` requires the audit's lighthouse result to have `status="pass"` or `"warn"`** — failed audits don't write (would overwrite good scores with garbage). The current implementation throws if lighthouse status is fail; that's intentional.
- **Tests should NOT call real Airtable.** Use `makeFakeBase()` from Phase 2.1. If a test needs to verify a network call, stub `global.fetch`.
- **The `fromAirtableBase` decision about `repoUrl` is the one place this plan asks for human input.** Don't commit a guess — surface to Tucker before merging.
- **The webhook test in Phase 1.4 imports from a `.mts` file.** Vitest handles this fine if the project's `tsconfig.json` doesn't exclude it. If it does, switch the import to a relative path or change the test to read the file as a module.
