## Findings ‚Äî MEDIUM

**MEDIUM-A ‚Äî Timing-gate spam screen is bypassable with a future `ts`.** [src/forms/action.ts:116-119](../../src/forms/action.ts#L116) + [src/forms/client.ts:132-139](../../src/forms/client.ts#L132). `elapsedMs()` returns `now() - ts` with only a `ts <= 0` guard ‚Äî a bot that posts a **future** epoch makes `elapsedMs` negative. `screenSubmission`'s check is `elapsedMs >= 0 && elapsedMs < MIN_FILL_MS`, so a negative value fails `>= 0`, skips the too-fast branch, and returns `{ ok: true }` ‚Äî the MIN_FILL_MS gate is silently bypassed. **Fix:** clamp in `elapsedMs()` (`Math.max(0, now() - ts)`) so a future ts ‚Üí 0ms ‚Üí correctly trips the gate, or treat negative as too-fast in `screenSubmission`. Honeypot still screens independently; this only defeats the timing layer. Add a `screenSubmission({ elapsedMs: -1000 })` test.

**MEDIUM-B ‚Äî Domain audit false-pass: a stale `Cert days remaining` survives a DNS failure ‚Üí the autochecker ticks "Domain, DNS & SSL" for an unreachable site.** [src/reports/airtable/websites.ts:429](../../src/reports/airtable/websites.ts#L429). On `resolved=false`, `domainResultFromAudit` yields `certDaysRemaining: null`, but `domainFields` only writes the field **when non-null** ‚Äî so the previous value (e.g. 90) stays in Airtable while "Domain checked at" is stamped fresh. At draft time, [auto-tick.ts:181-191](../../src/reports/auto-tick.ts#L181) reads the stale non-null value, passes the freshness check, and emits `pass` for a site that's currently down. **Fix:** write `Cert days remaining` **unconditionally** (Airtable accepts `null` to clear it), so `domainEvidence` sees `null` ‚Üí correctly emits `fail`. This is exactly the autochecker-integrity class the system is built to avoid.

**MEDIUM-C ‚Äî `openDb()` runs two Turso round-trips (`runMigrations`) on every handler invocation; no warm cache.** [src/db/client.ts:40](../../src/db/client.ts#L40) + [migrate.ts:10-13](../../src/db/migrate.ts#L10). Every warm Netlify invocation re-runs `CREATE TABLE IF NOT EXISTS _migrations` + `SELECT id FROM _migrations` before any real query ‚Äî added latency on the form-ingest hot path and every dashboard GET. **Fix:** a module-level `let migrated: Promise<void> | null` (or `let migrationsApplied = false`) resolved once per warm process; longer-term, a deploy-time migration step so the hot path skips `runMigrations` entirely. (The `migrate.ts:8` comment "cheap (one indexed SELECT)" also undercounts ‚Äî it's two statements.)

**MEDIUM-D ‚Äî `listNewSubmissions` has no `LIMIT` ‚Äî the cockpit deserializes every unread submission fleet-wide on each render.** [src/db/submissions.ts:94-101](../../src/db/submissions.ts#L94). `fleet-homepage.mts` loads the full array; `fleet-cockpit.ts` iterates it twice (badge counts + strip). **MEDIUM-E ‚Äî Submissions search doesn't escape `%` / `_` LIKE metacharacters.** [src/db/submissions.ts:156](../../src/db/submissions.ts#L156). `like = \`%${q.toLowerCase()}%\`` with no escape + no `ESCAPE` clause. Searching `_` matches any single char; `%` matches everything; `john_doe` matches `johnXdoe`. Parameterized, so **no injection** ‚Äî but silently wrong results on the operator's new search box. **Fix:** `.replace(/%/g,'\\%').replace(/_/g,'\\_')` + `ESCAPE '\\'` on each LIKE (needs the `sql` template form rather than `eb.fn('lower',‚Ä¶)+'like'`).
