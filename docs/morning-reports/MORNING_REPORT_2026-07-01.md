## Findings — MEDIUM

**MEDIUM-A — The "bad frequency" guard rail is at the wrong layer; the warning is dead code and two tests assert the opposite of production.** [src/reports/airtable/websites.ts:213-217](../../src/reports/airtable/websites.ts#L213) (`toFrequency`) + [:236-237](../../src/reports/airtable/websites.ts#L236) (reads the misspelled live key `"maintenence freq"`) vs [src/reports/due.ts:84](../../src/reports/due.ts#L84),[:120-135](../../src/reports/due.ts#L120) + [tests/reports/due.test.ts:206-221](../../tests/reports/due.test.ts#L206). `mapRow` runs every Airtable row through `toFrequency`, which returns `"None"` for any string not *exactly* in `FREQUENCIES` — a typo, a renamed select option, or a **trailing-space** `"Quarterly "` — so by the time `findDueReports`/`nextDueDate` see the value it's already been flattened. Consequences: (1) the `⚠ … unrecognized frequency` warn ([due.ts:131](../../src/reports/due.ts#L131), added by #197 precisely to stop a silent drop) **can never fire against live data** — the bad value was already turned into `"None"` upstream, so the site silently vanishes from scheduling with no signal; (2) the `.trim()` normalization is moot — `"Quarterly "` does **not** schedule in production; (3) [due.test.ts:210-221](../../tests/reports/due.test.ts#L210) asserts `"Quarterly "` schedules and [:206-207](../../tests/reports/due.test.ts#L206) asserts the warn fires, both **green only because the test factory (`makeWebsiteRow`) bypasses `mapRow`** — they assert the opposite of shipped behavior. **Why it's MEDIUM not HIGH:** the field is a **singleSelect** (verified live: choices `None/Yearly/Quarterly/Monthly`, exact-matching the code), so an operator can't free-type a typo today — the failure is *latent*, triggered only by an option rename/add or a trailing-space option. But this fleet has hit trailing-space Airtable values before (Vineyard Name), and a silent fleet-member unschedule is exactly the outcome #197 was written to prevent. **Fix:** move the LOUD warning into `toFrequency` (warn once at the read boundary on a non-empty unrecognized string, then return `"None"`), delete the now-unreachable warn + `.trim()` branches in `due.ts`, and rewrite the two tests to feed a raw Airtable-shaped record through `mapRow`. _(Confidence: high — both code paths traced; the test-factory bypass confirmed by reading `makeWebsiteRow`; field type confirmed against the live schema.)_
